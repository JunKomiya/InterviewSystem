# AI面接練習システム 実行手順書
---

## 1. システムの概要
本システムは、学生が自立して面接練習を行えるように開発された、Streamlitベースのデスクトップ・ウェブアプリケーションです。
カメラ画像から表情や視線を読み取り、AI面接官と対話しながら面接練習を行うことができます。

### 主な機能
* **AI面接官（対話機能）**: Gemini API（または事前に定義されたモック）を利用し、面接の質問や深掘りを対話形式で行います。
* **音声合成（TTS）**: `edge-tts` を使用し、面接官が自然な音声で質問を読み上げます。
* **視線・表情トラッキング**: PC内蔵または外付けWebカメラを用いて、面接練習中の視線の安定度やまばたき、表情（笑顔など）をリアルタイムに検知・記録します。
* **評価レポート**: 面接終了後、対話の質（回答の一貫性・適切さ）と非言語情報（視線や表情の安定度）を総合的に分析し、スコア化およびフィードバックを行います。

## 2. 稼働環境と前提条件
本システムを動作させるためには、以下の環境が必要です。

### 動作環境
* **OS**: Windows 10 / Windows 11
* **Python**: **Python 3.12** （必須）
  > **【注意】** Windows環境下における `MediaPipe` ライブラリの互換性の都合上、Python 3.13等では動作しません。必ず **Python 3.12** をご用意ください。また、本プロジェクトでは仮想環境（`.venv`）を使用せず、グローバル環境で動作させる構成となっております。
* **ハードウェア**:
  * Webカメラ（顔・表情・視線のトラッキング用）
  * スピーカー または イヤホン・ヘッドホン（面接官の音声再生用）
* **通信環境**: インターネット接続（Gemini APIの呼び出し、および `edge-tts` による音声合成の生成に必要です）

### APIキーの準備
AI面接官機能および評価分析を行うために、Google AI Studio 等から **`GEMINI_API_KEY`** を取得しておく必要があります。
※APIキーが設定されていない、または無効な場合は、自動的に「MOCKモード（仮面接官による練習）」に切り替わって動作します。
※提出先やご自身で実行する際は、必要に応じてGoogle AI Studioから取得したAPIキーを環境変数 GEMINI_API_KEY に設定してご利用ください。（未設定時は自動でモックモードにて起動します）

## 3. セットアップ手順

以下の手順で環境構築を行います。

### 1. Python 3.12 のインストール確認
PowerShellまたはコマンドプロンプトを開き、以下のコマンドで Python 3.12 がインストールされているか確認します。
```bash
python3.12 --version
```
※ `Python 3.12.x` と表示されれば問題ありません。

### 2. 依存ライブラリのインストール
以下のコマンドを実行し、必要な外部パッケージをインストールします。必ず `python3.12` を明示的に指定して実行してください。
```bash
python3.12 -m pip install streamlit mediapipe==0.10.14 opencv-python edge-tts google-genai numpy pandas openpyxl
```

## 4. 起動手順

Windows環境において、`MediaPipe` が `protobuf` と競合してエラーになる現象（`Failed to parse: node ...`）を回避するため、起動時に特定の環境変数を適用する必要があります。
そのため、本システムでは専用の起動スクリプトを用意しています。

### 推奨される起動方法
プロジェクトのルートディレクトリにある以下のいずれかのファイルをダブルクリック、または実行してください。

* **`run.bat`** (Windowsバッチファイル) をダブルクリックする
* **`run.ps1`** (PowerShellスクリプト) をPowerShellで実行する

これらのスクリプトを実行すると、内部で環境変数 `PROTOCOL_BUFFERS_PYTHON_IMPLEMENTATION=python` を自動的にセットした上で、Streamlitが自動起動します。

#### 手動で起動する場合（コマンド実行）
ターミナルから手動で起動したい場合は、以下の手順でコマンドを実行してください。

**PowerShellの場合:**
```powershell
$env:PROTOCOL_BUFFERS_PYTHON_IMPLEMENTATION="python"
python3.12 -m streamlit run main.py
```

**コマンドプロンプト(cmd.exe)の場合:**
```cmd
set PROTOCOL_BUFFERS_PYTHON_IMPLEMENTATION=python
python3.12 -m streamlit run main.py
```

## 5. システムの操作手順（面接練習の流れ）

Streamlitが正常に起動すると、既定のWebブラウザが自動的に立ち上がり、ローカルホスト（通常は `http://localhost:8501`）にシステム画面が表示されます。以下の流れに沿って操作します。

### ステップ 1: スタート画面
1. 画面上の「面接練習を開始する」ボタンをクリックします。
2. 過去に実施した面接履歴がある場合、画面下部でその結果やスコアのトレンドを確認できます。

### ステップ 2: 面接設定およびキャリブレーション（SETUP）
1. **プロフィールの入力**: 氏名を入力します。
2. **面接条件の設定**:
   * 志望する「業界」や「職種」を選択します（Excelファイル等の情報を参照することも可能です）。
   * 「模擬面接官（AI面接官 または モック）」を選択します。
3. **カメラテストおよび視線キャリブレーション**:
   * 接続されているカメラを選択し、「映像を開始」します。プレビューで自身の顔が映るか確認します。
   * 正確な視線トラッキングを行うために、「📐 視線キャリブレーション」を実行します。画面に表示されるポイントを指示に従って見つめることで、初期位置の測定が行われます。
4. **準備完了**: 設定が完了したら「面接室へ移動する」ボタンを押します。

### ステップ 3: 面接本番（INTERVIEW）
1. **質問の開始**: 面接官（ナナミ）が音声合成によって自動で質問を読み上げ、チャットログに質問が表示されます。
2. **回答の入力**: 回答欄（タイピング入力）に自分の答えを入力し、「回答を送信する」をクリックします。
3. **対話の継続**:
   * 回答を送信すると、それに対する面接官からの返答や、追加の深掘り質問がなされます。
   * 同様に回答を繰り返します。
   * *※注意: 面接中はカメラの目線を意識し、背筋を伸ばして面接官の画像やカメラを見つめるように心がけてください。裏で視線トラッキングが作動しています。*

### ステップ 4: 面接評価（EVALUATION）
面接が全て終了すると、自動的に評価レポート画面に遷移します。
1. **総合診断結果**: 100点満点中何点か、およびランク（A, B, Cなど）が判定されます。
2. **回答内容の評価 (AI分析)**: 「回答の一貫性」や「回答の適切さ」がパーセンテージで表示され、過去の平均データとの比較ゲージも表示されます。
3. **非言語情報の分析**: 面接中の視線トラッキングデータを元に、「視線の安定度」「笑顔の割合」などが詳細なグラフや数値としてフィードバックされます。
4. **次回へのアドバイス**: Gemini API による多角的なフィードバックコメントが表示されます。
5. **練習の終了**: 「トップに戻る」ボタンを押すと、初期画面へ戻ります。

## 6. トラブルシューティング

* **「Failed to parse: node { calculator: 'ImagePropertiesCalculator' ... }」というエラーが出る場合**
  * **原因**: Windows環境下で MediaPipe と Protobuf のバージョン整合性が取れていない場合に発生します。
  * **対策**: `run.bat` または `run.ps1` からシステムを起動しているか確認してください。手動で起動する場合は必ず環境変数 `PROTOCOL_BUFFERS_PYTHON_IMPLEMENTATION=python` が設定されていることを確認してください。
* **カメラ映像が真っ暗、またはエラーになる場合**
  * **対策**:
    1. カメラが別のアプリ（Teams, Zoom, Discord等）で使用中になっていないか確認してください。
    2. 面接設定画面で正しいカメラ（Device ID）が選択されているか確認し、「映像を再起動」を試してください。
* **音声が再生されない場合**
  * **対策**:
    1. PCのスピーカー音量やミュート設定を確認してください。
    2. `edge-tts` は音声生成時にインターネットを介します。ネットワークが切断されていないか確認してください。
* **Gemini API のエラー、または動作が遅い場合**
  * **対策**: APIキーが正しいか確認してください。一時的な接続エラーの場合は、設定画面で「モック面接官」を選択して動作確認を行うことができます。
